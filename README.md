## Стек

| Слой     | Технологии                        |
|----------|-----------------------------------|
| Frontend | React 18, TypeScript, Vite        |
| API      | Express, TypeScript, tsx          |
| Backend  | Express, TypeScript, tsx          |

## Структура проекта

```
apps/
├── frontend/   — React-приложение
├── api/        — Прокси-сервер
└── backend/    — Express API + база данных
```

---

### 1. Backend (`apps/backend`)

В файле `src/index.ts`:
- Подними Express-сервер
- Опиши тип `User`
- Подключись к «базе данных» (`src/db/users.json`)
- Реализуй эндпоинт `GET /users`, который возвращает список пользователей

**Порт:** `3001`

---

### 2. API (`apps/api`)

В файле `src/index.ts`:
- Подними Express-сервер
- Напиши прокси, который перенаправляет запросы с `/api/*` на Backend (`http://localhost:3001`)

**Порт:** `3002`

---

### 3. Frontend (`apps/frontend`)

В файле `src/App.tsx`:
- Опиши тип `User`
- Сделай запрос к API-серверу (`http://localhost:3002/api/users`)
- Выведи полученных пользователей в виде таблицы со столбцами: **ID**, **Имя**, **Email**, **Роль**

---

### 4. Auth-сервис (`apps/auth`)

- Реализуй `POST /auth/login` — принимает `email`, возвращает JWT-токен
- Backend должен проверять токен перед тем, как отдать список пользователей
- Фронт должен логиниться, сохранять токен и передавать его в заголовке каждого запроса

**Порт:** `3003`

---

### 5. API Key

- Фронт должен передавать `x-api-key` в заголовке каждого запроса
- Ключ должен валидироваться до того, как запрос дойдёт до Backend

---

## Запуск

```bash
# Из корня проекта — устанавливаем зависимости
npm install

# Запускаем каждый сервис в отдельном терминале
npm run backend
npm run api
npm run frontend
```

---

## Требования

- Не менять структуру папок
- Не добавлять лишние библиотеки (только то, что уже есть в `package.json`)
- Типизация обязательна — `any` не допускается
